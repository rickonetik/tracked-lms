# Story 0.9 — CI pipeline — Проверка состояния git

## Проверка файлов в git

Проверяю что все необходимые файлы добавлены в git репозиторий.

## Что должно быть в git

✅ `package.json` - корневой
✅ `turbo.json`
✅ `tsconfig.json`
✅ `apps/api/package.json`
✅ `apps/webapp/package.json`
✅ `apps/bot/package.json`
✅ `apps/*/src/` - исходные файлы
✅ `.github/workflows/ci.yml`

## Если CI всё ещё падает

Возможные причины:
1. CI не перезапустился после последнего push
2. Файлы не полностью добавлены
3. Другая ошибка в логах

**Нужно:**
1. Проверить логи CI в PR
2. Убедиться что CI перезапустился после последнего push
3. Если ошибка та же (package.json not found), значит файлы не попали в git

## Следующие шаги

1. Обновить PR страницу
2. Проверить что CI перезапустился
3. Посмотреть новые логи ошибок
4. Прислать новые ошибки если они есть
